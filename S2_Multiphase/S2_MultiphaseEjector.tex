\subsection{Python virtual environment}

To facilitate the generation of the mesh with \terminalcmd{blockMesh} we are going to 
use a python code. It is convenient to create first a user virtual environment, so
that packages can be locally installed. Here follows the procedure

\begin{lstlisting}[style=terminal]
# Create the virtual environment. Here the name is 'my_env' but it can be any name
python -m venv my_env
# Activate the virtual environment
source ./my_env/bin/activate
# From now on a '(my_env)' will be shown with prompt
# To deactivate the virtual environment, just type
deactivate 
\end{lstlisting}

The module 
\href{https://github.com/takaakiaoki/ofblockmeshdicthelper}{ofblockmeshdicthelper}, by 
A. Takaaki will be used. It has to be installed in the environment with

\begin{lstlisting}[style=terminal]
pip install 'ofblockmeshdicthelper @git+https://github.com/takaakiaoki/ofblockmeshdicthelper.git'
\end{lstlisting}

If some other module is needed, just install it with \terminalcmd{pip install <module name>}.

\subsection{Running the Simulation}

The simulation process is split into two automated phases: mesh generation and solver
execution. Both phases are implemented in shell scripts, to ensure consistent
and reproducible workflow.

\subsubsection{Mesh generation}

The first step is to create the \filename{system/blockMeshDict} file. It is
done with the python code \filename{blockMeshDictGen.py}, that load the 
\terminalcmd{ofblockmeshdicthelper} mentioned in the previous section.


Check that the python virtual environment is activated and run
\begin{lstlisting}[style=terminal]
python blockMeshDictGen.py
\end{lstlisting}
The file \filename{system/blockMeshDict} has been created.

Run the script for the mesh generation

\begin{lstlisting}[style=terminal]
./genMesh.scr
\end{lstlisting}

\warning{It the command fails, it is probable that it is not executable. Change it 
with}
\begin{lstlisting}[style=terminal]
chmod +x ./genMesh.scr
\end{lstlisting}

Have a look at the script and discuss the meaning of each command. Check the output
of the \terminalcmd{checkMesh} command and 
visualize the mesh with \terminalcmd{paraFoam}

\subsubsection{Initialization and solver execution}

The \filename{runCase.scr} performs several tasks:

\begin{enumerate}
	\item It copies the content of the folder \filename{0.orig} to the forlde 
	\filename{0}.
	\item It initializes the phase distribution with the command 
	\terminalcmd{setFields} and the dictionary \filename{system/setFieldsDict}
	\item It runs the \terminalcmd{interFoam} solver. 
\end{enumerate}

The simulation will take about 10 minutes. In the meanwhile the configuration files of
the case can be studied:

\begin{itemize}
	\item \filename{system/controlDict} and functions definitions
	\item \filename{0.orig/*}
	\item \filename{constant/*}
	\item \filename{system/fvSolution} and \filename{system/fvSchemes}
\end{itemize}

Note that the flow rate in main secondary inlets are\footnote{These are the flow rate
for \ofkeyword{wedge} of the mesh. For the complete flow rate, these numbers have to 
be multiplied by the proportionality factor od total and wedge sections.}
\begin{eqnarray*}
	Q_s &=& \qty{6.35e-5}{m^3/s} 
	\\
	Q_m &=& \qty{5.58e-5}{m^3/s}
\end{eqnarray*}
According to that the dimensionless \textbf{entrainment ratio} is
\begin{equation*}
	M =  \frac{Q_s}{Q_m} = 1.14
\end{equation*}

The case can be also downloaded from the GitHub repository.

\subsection{Running the simulation}
The step for mesh generation, setup and running of the solver are exactly the same. There are 
some key differences on the setup.

\begin{itemize}
	\item \textbf{Phase interaction description}. In VOF there is only a configuration file with surface
	tension. In Euler-Euler the interaction between forces has to be completely modeled 
	through the \filename{constant/phaseProperties} dictionary. For a detailed description of all the items, the great Master Thesis by Otromke\footnote{Otromke, Malte. 2013. “Implementation and Comparison of Correlations for Interfacial Forces in a Gas-Liquid System within an Euler-Euler Framework.”} can be consulted.
	\item \textbf{Fields files and initial condition}s. The main characteristic of Euler-Euler 
	approach is that fluid flow equations are \textbf{solved for both fluids}. That
	mains that two sets of field files are needed. For instance, there is a \filename{0/U.water} file but also a \filename{0/U.air} file.
	\item \textbf{Turbulence modeling}. Likewise, two turbulence modeling files are needed, one
	for each phase. They can be different, but compatible.
\end{itemize}

\section{Student Exercises and Extensions}

\subsection{Basic exercises for Part A: VOF}

\begin{enumerate}
	\item Effect of Inlet Velocity
	
	Modify the water inlet velocity from \qty{2}{m/s} to \qty{4}{m/s}. How does
	entrainment ratio change? Is the relationship linear?
	
	\item Effect of Surface Tension
	
	Set surface tension to zero. How does the interface shape change? How does $M$ 
	change? 
\end{enumerate}

\subsection{Basic exercises for Part B: Euler-Euler}

\begin{enumerate}
	\item Drag Model comparison
	
	Change drag model for \ofkeyword{(air in water)} from 
	\ofkeyword{SchillerNaumann} to \ofkeyword{TomiyamaCorrelated}. A 
	value of 16 will be need in the keyword \ofkeyword{A} (see Master Thesis
	by Otromke)
	
	\item Turbulent Dispersion
	
	In this tutorial, turbulent dispersion has not been considered. However, it is
	an important factor for high Reynolds number flows. Include the usual Burn's 
	model, with a Schmidt number $\sigma=1.0$.
	
\end{enumerate}
